android屏幕适配

  • 屏幕适配
  • 随笔

　　由于android系统的开源，使得任意一家手机厂商都能拿到android源码并按照自己的想法来生产Android手机。这样就带来了一个非常严重的问题，那就是android碎片化非常严重(具体数据可以参考友盟的统计)。因此在我们进行android开发时需要考虑屏幕适配…

1、前言

　　先说一点废话，其实我有时会想PC端的碎片化其实比android都还严重，但是做web前端的可以根据百分比来设置宽高从而不用适配，那Android为什么也不通过这种方式来解决屏幕适配啊（听说android5.0以后已经支持用百分比来设置宽高，还有一些开源库也可以通过百分比来实现）？好了，我唠叨完了，哈哈。下面就来说说在android里面如何进行适配。

2、如何适配？

　　首先开发时我们要基于主流屏幕来开发（比如现在的主流屏幕是1280*720）,要遵循不用AbsoluteLayout(绝对布局), 多用相对布局&线性布局(权重), 要用dp,不用px的原则。这样基本上就没有大问题了。但是如果出问题了呢？就用下面五种方式来进行处理：

图片适配： 就是我们根据不同分辨率让美工切不同的图从而放在不同分辨率的文件夹（mdpi,hdpi,xdpi,xxdpi）下，但是由于我们一般只用一套图，因此此方法用的不多。
布局适配： 根据不同分辨率来写不同的布局，如要适配480x800则需要在res文件下创建layout-800x480这样的文件夹，然后在其文件夹下写相应的布局。此方法用的也不多。
尺寸适配： 根据 dp = px/设备密度 这个公式来将px转化成dp，因此我们会根据需要将一些需要适配的长度放在 dimens.xml 文件夹下定义，然后根据分辨率来创建相应的value文件夹，如需适配1280x720则创建values-1280x720文件夹，因此此方法用的较多。
权重适配： 主要用于线性布局，有点类似百分比来设置宽高的味道。我们可以通过 android:weightSum=”3” 来设置权重总数，局限性有点大
代码适配： 获取屏幕宽高,动态计算控件大小，很少用

3、如何将px与dp之间进行转换？

/**
 * dp转px
 */
public static int dp2px(Context ctx, float dp) {
    //获取当前设备的密度
    float density = ctx.getResources().getDisplayMetrics().density;
    int px = (int) (dp * density + 0.5f);// 这里加上0.5f主要是为了提高精度

    return px;
}

//px转dp
public static float px2dp(Context ctx, int px) {
    float density = ctx.getResources().getDisplayMetrics().density;
    float dp = px / density;
    return dp;
}

通过上述方法我们就可以进行对android进行适配啦，如有不正之处，请指出。谢谢！
