面向holder编程

  • android
  • 面向holder

　　我在这里说到面向holder编程，或许有人会问什么是面向holder啊？其实面向holder就是把一个页面的能拆分的布局都当做一个holder，然后每个人就来做这样的一个holder，从而加快项目的开发进度。也让代码更加清爽、简洁、便于管理…

1、引言

　　说到Holder，或或许有一种熟悉感，但是又不知道在哪里用过，但是仔细想想就会发现其实我们在ListView或者GridView的adapter里面就用到了Holder的。下面就来回忆一下是怎么使用的。

2、Adapter的封装

　　一般我们用adapter时，会发现getCount()、getItem(int position)、getItemId(int position)这三个方法的代码几乎一样，因此我们就将这些代码提出来封装成一个单独的类。代码如下：

public abstract class MyBaseAdapter<T> extends BaseAdapter {
protected Context context;
public List<T> mList;
protected LayoutInflater inflater;

public MyBaseAdapter(Context context, List<T> mList) {
    this.context = context;
    this.mList = mList;
    inflater = (LayoutInflater) context
            .getSystemService(Context.LAYOUT_INFLATER_SERVICE);
}

@Override
public int getCount() {
    return mList.size();
}

@Override
public Object getItem(int position) {
    return mList.get(position);
}

@Override
public long getItemId(int position) {
    return position;
}

@Override
public abstract View getView(int position, View convertView,
                             ViewGroup parent);

//更新adapter
public void setList(List<T> mList) {
    this.mList = mList;
    notifyDataSetChanged();
}
}

　　然后在每次使用adapter时再来创建Holder，然后再在getView(int position, View convertView, ViewGroup parent)的具体实现方法中来调用Holder。但是一个项目的adapter比较多时，不停的创建holder跟实现getView(int position, View convertView, ViewGroup parent)就很麻烦、那么我们能不能进一步的封装呢？那么就请看下面。

3、adapter的进一步封装

　　首先我来说一下，getView(int position, View convertView, ViewGroup parent)里面的convertView就是缓存的布局，也就是我们可以把这个布局的加载放在自己写的Holder里面去。然后也就可以把对于convertView布局及其控件写到Holder里面去，然后getView(int position, View convertView, ViewGroup parent)就只是返回一个View就行了。然后在把Holder里面的相同的代码提取出来（这里说的比较抽象）。我们先来看看代码吧，将getView(int position, View convertView, ViewGroup parent)也封装起来。代码如下：

public abstract class MyBaseAdapter<Data> extends BaseAdapter {
private Context context;
private List<Data> list;

public MyBaseAdapter(List<Data> list) {
    this.list = list;
}

@Override
public int getCount() {
    return list.size();
}

@Override
public Object getItem(int position) {
    return list.get(position);
}

@Override
public long getItemId(int position) {
    return position;
}

public void setList(List<Data> list) {
    this.list = list;
    notifyDataSetChanged();
}

@Override
public View getView(int position, View convertView,
        ViewGroup parent) {
    BaseHolder<Data> holder;
    if (convertView == null) {
        holder = getHolder();
    } else {
        holder = (BaseHolder) convertView.getTag();
    }
    Data str = list.get(position);
    holder.setData(str);
    return holder.getView();
}

public abstract BaseHolder<Data> getHolder();

}

　　这里的Data表示泛型（如果不懂泛型请看Java基础）。然后我们在写一个BaseHolder的类，代码如下：

public abstract class BaseHolder<Data> {
private View convertView;

public BaseHolder() {
    convertView=initView();
    convertView.setTag(this);
}

public void setData(Data data) {
    refreshView(data);
}
public View getView() {
    return convertView;
}
//将数据给相应的控件
public abstract void refreshView(Data data);

//初始化控件
public abstract View initView();
}

　　这样我们将adapter封装好了，我们就来看一下调用这些类的代码：

private class MyAdapter extends MyBaseAdapter<String> {

    public MyAdapter(List<String> list) {
        super(list);
    }

    @Override
    public BaseHolder<String> getHolder() {
        return new ViewHolder();
    }
}

private class ViewHolder extends BaseHolder<String> {
    private TextView text;
    private ImageView image;

    @Override
    public void refreshView(String str) {
        this.text.setText(str);
    }

    @Override
    public View initView() {
        View convertView = View.inflate(getApplicationContext(),
                R.layout.item_list, null);
        this.image = (ImageView) convertView
                .findViewById(R.id.item_list_img);
        this.text = (TextView) convertView
                .findViewById(R.id.item_list_text);
        return convertView;
    }
}

　　这样我们是不是就只需要操作ViewHolder这个类了啊，就很方便了，哈哈 　　

4、面向Holder编程

　　但是仅仅是因为能将adapter进一步封装还吸引不了我的注意力，并且我认为第一步的封装已经很不错了。但是我为什么还要说呢？是因为这里有一个 面向Holder的思想 。这里的Holder不仅仅局限于adapter里面了。因为Holder主要是加载布局跟赋值，那么我们就可以把一个页面看做由若干个Holder组成。这样我们就能把一个复杂的页面交给多个人做，从而来提升效率，也能使一个项目的分工更明确，出现的问题更少。这就是面向Holder，也能避免一些菜鸟乱改代码。
